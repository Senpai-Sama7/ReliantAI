State Machines

ClearDesk document flow

UPLOADED→ OCR + Claude → PARSED→ conf ≥ 0.70 + amount < limit → READY→ EXPORTED
PARSED→ conf < 0.70 OR amount ≥ threshold → ESCALATED→ human approve → READY
Security: Document preview MUST pass DOMPurify before DOM insertion. Never use innerHTML with raw document bytes.

Money dispatch + LIFE_SAFETY protocol

INBOUND→ triage_agent → LIFE_SAFETY→ 911 directive · NO dispatch · owner notified
INBOUND→ triage_agent → PENDING→ SCHEDULED→ EN_ROUTE→ COMPLETED
Security invariant #3: Gas/CO/smoke/fire keywords MUST trigger 911 directive. UI must show LIFE_SAFETY banner, disable dispatch actions, and display "no technician dispatched" — even if backend check fails.

Security Invariants

These MUST be enforced in every implementation. All 7 appear in Settings → Security in the console.

#InvariantUI enforcement
1No auth tokens in browser storagelocalStorage / sessionStorage / IndexedDB must never contain JWTs or API keys. Session state is server-side only.
2Document preview is text-safeAll document content must pass DOMPurify.sanitize() before DOM insertion. No innerHTML with raw bytes.
3Life-safety auto-dispatch blockedLIFE_SAFETY → show red pulse banner, disable dispatch actions, display "no technician dispatched" message.
4No default credentialsSettings UI must reject empty API key fields. All services require explicit env vars with :?required syntax.
5No raw UUID KV accessClearDesk cross-device sync uses signed session codes (6-char, short-lived) — never raw document UUIDs.
6Fail-closed authIf /auth/health fails, UI must block access and show error. Never fall through to an unauthenticated state.
7No direct service-to-service callsAll cross-service data flows through the integration event bus or Kong gateway. No direct API calls between service UIs.

Implementation Checklist

  • Tokens imported from colors_and_type.css — no hardcoded hex values in components
  • All data values use IBM Plex Mono — no sans-serif for numeric / ID fields
  • Border-radius max 2px — no rounded-lg, rounded-xl, or rounded-full in Tailwind
  • No box-shadows on cards/panels — elevation via background-color stepping only
  • Hover: background steps up one level (--re-surface → --re-surface-hi)
  • Loading states: shimmer skeleton (Skel component) matching screen layout
  • Empty states: centered, lucide icon + action CTA (EmptyState component)
  • LIFE_SAFETY banner: animation: pulse 1.4s ease infinite — red, full-width, sticky top
  • Confidence scores: ConfBar — thresholds at 0.90 (green), 0.70 (gold), below (red)
  • Event types in snake.dot.notation — never camelCase in UI labels
  • Status enums in SCREAMING_SNAKE_CASE — never lowercase in badges
  • All cross-service navigation via event bus — no direct service-to-service API calls
  • SSE connections use useEventStream hook — always clean up on unmount
  • Command palette (⌘K) available on every authenticated screen
